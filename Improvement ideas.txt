IMPROVEMENT IDEAS — XAG/USD HFT Alpha Research
================================================

[DONE] 1. Fix Time-Series Label Leakage
   Labels are forward returns, so training-set tail samples' labels overlap with the
   validation window's future prices. This causes look-ahead bias in rolling backtests.
   Fix: add an embargo gap between train and validation sets equal to the prediction
   horizon h (e.g., gap=50 for trend_50m).
   → Implemented: per-horizon embargo gaps added to rolling_backtest().

[DONE] 2. Use Better Evaluation Metrics
   Accuracy/Precision/Recall/F1 are misleading for high-frequency directional prediction.
   Preferred metrics:
   - Log Loss (probability calibration quality)
   - Brier Score (mean squared error of probabilities)
   - AUC-ROC / PR-AUC (ranking/discrimination ability)
   - Post-cost strategy return, Sharpe ratio, max drawdown
   Note: 51% accuracy with better calibration often outperforms 55% in live trading.
   → Implemented: Log Loss, Brier Score, and AUC-ROC now computed for all models.

[DONE] 3. Improve Label Definition (Dead-Zone Labels)
   Binary ret>0 labels treat tiny noise moves as valid signals. Switching to a
   three-class dead-zone scheme reduces false signals:
   - Neutral (no trade): |return| below a threshold (e.g., 0.2× rolling volatility)
   - Only trade clearly directional moves
   This step typically outperforms any model change.
   → Implemented: three-class labels (Down / Noise / Up) via train-set terciles.

[DONE] 4. Increase Rolling Window Size
   Previous settings (training=500, validation=100, step=500) caused:
   - Short training windows with high variance
   - Infrequent model updates
   Recommended: training=2000–5000, validation=200–500, step=50–200.
   → Implemented: training_size=1800, validation_size=200, step_size=1800.

[DONE] 5. Strengthen Temporal Feature Structure
   The base microstructure factors are solid, but temporal context improves signal:
   - Factor lag terms (lag1, lag5, lag20)
   - Factor rate-of-change (diff/return)
   - Rolling mean and rolling volatility (regime features)
   - Session features (hour-of-day, Asia/Europe/US session flags)
   - Volatility state features (low/mid/high vol regimes)
   → Implemented: 20 base factors expanded to 162 features with all of the above.

[DONE] 6. Refine Tree Model Pipeline
   Tree models should skip PCA (tree splits are already non-linear). Key additions:
   - Probability threshold tuning (avoid hardcoded 0.5)
   - Probability calibration (Platt scaling / Isotonic regression)
   → Partially done: PCA removed for tree models; custom thresholds applied per model
     (Bagging=0.50, Lasso/LDA=0.65, GRU=0.40). Probability calibration not yet applied.

[DONE] 7. Add Ensemble Model
   Combine predictions from complementary models to improve accuracy and signal quality.
   → Implemented: equal-weight probability averaging across Lasso + LDA + Bagging.
     Ensemble achieves highest accuracy (51.19%) and best win rate (58.78%) in backtest.
     Cohen's Kappa analysis confirms RNN has lowest agreement with tree/linear models
     (Kappa ~0.40), making it a strong candidate for future ensemble inclusion.

OPEN IDEAS — Not Yet Implemented
---------------------------------

8. Add Realistic Baselines
   Two simple baselines should be benchmarked before claiming model value:
   - Always-predict-up: buy-and-hold equivalent at signal level
   - Momentum baseline: predict same direction as last minute's return
   A model only has live-trading value if it consistently beats both.

9. Probability Calibration
   Models like KNN and Naive Bayes are significantly miscalibrated (Brier > 0.74).
   Even well-performing models (GRU, Bagging) would benefit from post-hoc calibration:
   - Platt scaling (logistic regression on raw probabilities)
   - Isotonic regression (non-parametric, better for large datasets)
   Calibrated probabilities improve threshold-based signal construction and risk sizing.

10. Threshold Optimization per Model
    Current thresholds (0.40–0.65) were set manually. A more principled approach:
    - Grid search over thresholds on a held-out validation window
    - Optimize for Sharpe ratio or Calmar ratio (not accuracy)
    - GRU at 0.40 generates 458 trades with low returns — raising the threshold
      should filter noise and improve risk-adjusted performance.

11. Cross-Asset Validation
    All 162 features and the rolling backtest framework are asset-agnostic.
    Test whether XAG factors generalize to:
    - Other commodities: XAU/USD (gold)
    - Crypto: BTC
    - FX: EUR/USD, GBP/USD
    - Equity: AAPL, QQQ
    Strong cross-asset performance would validate the factor construction approach.

12. Expand Ensemble to Include GRU
    Current ensemble (Lasso + LDA + Bagging) skips GRU due to different training cadence.
    Options:
    - Meta-learner (stacking): train a logistic regression on out-of-fold predictions
      from all four model families as input features
    - Diversity-weighted ensemble: weight by inverse average Kappa (GRU gets highest
      weight due to lowest agreement with others)
    GRU's high AUC (0.5633) and low Kappa with tree models (~0.40) make it a strong
    complement in an ensemble.

13. GRU at Longer Horizons
    GRU is currently only evaluated at the 5-minute horizon due to training time (~88 min
    per full backtest). The 50m horizon shows the strongest RankIC for mean-reversion
    factors (MR IC-IR = -0.631), which may suit the GRU's sequential modeling better.
    Consider training a single GRU with multi-task output heads for all four horizons.

14. Signal Quality Decay Analysis
    Accuracy degrades from ~51% at 5m to ~44% at 50m. Investigate:
    - Whether factor IC decay rate explains model accuracy decay
    - Whether different factor subsets should be used at different horizons
    - Whether the dead-zone threshold should scale with horizon

15. Live Paper Trading Simulation
    The backtest assumes perfect execution at bar close. A more realistic simulation:
    - Introduce execution delay (signal at bar close, fill at next bar open)
    - Model bid-ask spread as a function of volatility regime
    - Track rolling Sharpe in real-time to detect regime shifts requiring model retraining
